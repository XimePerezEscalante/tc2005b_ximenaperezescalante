Preguntas

  1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

    2. De acuerdo con la BBC, Internet es una red de computadoras que están conectadas entre sí en todo el mundo.
    Mientras que World Wide Web es una colección gigantezca de páginas que se encuentran en esa red de computadoras.
    Por lo tanto, usamos internet para acceder a la World Wide Web.

  2. ¿Cuáles son las partes de un URL?

    3. Según IBM, se forma de tres a cuatro componentes:

    • Un esquema: identifica el protocolo que se utilizará para acceder al recurso en Internet. Puede ser HTTP o HTTPS.
    • Un nombre de host: identifica el host que tiene el recurso y también pueden estar seguidos de un número de puerto.
    • Un camino: identifica el recurso específico en el host al que el cliente web quiere acceder.
    • Una cadena de consulta: En caso de ser utilizada, suele ser una cadena de pares de nombre y valor, que son separados por un ampersand (&)
  3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
    • GET: solicita una representación de un recurso específico. Sólo puede recuperar datos.
    • HEAD: pide una respuesta idéntica a la de una petición GET, pero sin el cuerpo de la respuesta.
    • POST: envía una entidad a un recurso en específico, causando a menudo un cambio en el estado o efectos secundarios en el servidor.
    • PUT: reemplaza todas las representaciones actuales del recurso de destino con la carga útil de la petición.
    • PATCH: aplica modificaciones parciales a un recurso.
    • DELETE: borra un recurso en específico.
  4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    5. Se debe usar el método POST porque así no se ven los datos en el URL para que una persona externa se los pueda robar.

  5. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    6. Este código de respuesta de estado satisfactorio, también conocido como OK, significa que una solicitud ha tenido éxito. Por lo que no representa ningún error.

  6. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    7. Este error significa Page Not Found, de acuerdo con Google, dependiendo de la causa se debe considerar corregirlo o no.
    Por ejemplo, si ocurre por una página que ha sido eliminada completamente y no tiene un reemplazo o se trata de una URL que nunca ha existido en el sitio, no vale la pena cambiarlo.
    Sin embargo, si es una URL enviada para su indexación, vale la pena corregir el error.
    Finalmente, si se debe a un error de ortografía o escritura cometido frecuentemente por los usuarios, se recomienda mapear las palabras alternativas a la misma página del sitio usando redirecciones u otros mecanismos.

  7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    8. Sí, ya que su significado es Internal Server Error y ocurre cuando el servidor se encuentra una condición inesperada que le impide cumplir con la solicitud. A veces va acompañado de texto, pero en navegadores como Firefox y Safari, puede ser representado por una pantalla en blanco.

  9. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?
    • Introducción de etiquetas semánticas como <header>, <footer>, <section> y <article>.
    • Control de audio y video con las etiquetas <audio> y <video>.
    • Las aplicaciones web pueden almacenar datos localmente en el dispositivo del usuario gracias a la localStorage API.
    • Los sitios web pueden acceder a la ubicación geográfica de sus usuarios.
    • Uso de base de datos en SQL para almacenar datos Offline.

  13. ¿Cuál es el ciclo de vida de los sistemas de información?

    14. Un ciclo de vida engloba la infraestructura, la organización, el personal y todos los componentes necesarios para la recopilación, procesamiento, almacenamiento, transmisión, visualización, diseminación y organización de la información.
    El ciclo se compone de la siguiente manera:

    • Planificación
    • Análisis
    • Diseño
    • Implementación
    • Pruebas
    • Instalación o despliegue
    • Uso y mantenimiento
  14. ¿Cuál es el ciclo de desarrollo de sistemas de información?

    15. Este ciclo en cascada busca hacer las cosas bien desde el inicio y se conforma de la siguiente manera:

    • Definición de requerimientos
    • Análisis y Diseño del Software
    • Implementación y Prueba de unidades
    • Integración y Prueba del Sistema
    • Operación y Mantenimiento

  15. Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

    16. Debe usarse lo menos posible, idealmente nunca, debido a que denota una mala planeación del diseño.

  16. Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

    17. Porque se debe escoger una imagen de alta calidad, que sea coherente con el contenido de la página y que no signifique un mayor tiempo de carga.

  17. Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

    18. Se recomienda usar unidades relativas, más que absolutas, ya que estas últimas se ven diferentes desde nuestra perspectiva de acuerdo al dispositivo que se esté utilizando. Para tipografías es mejor usar em y para alinear texto con imágenes es mejor px.

  18. ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

    19. Porque se reduce el tamaño del archivo y se mandan menos bytes, haciendo que la página cargue mucho más rápido.
